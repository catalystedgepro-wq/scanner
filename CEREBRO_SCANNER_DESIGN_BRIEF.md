# Cerebro / Scanner Design Brief

## Owner

- Primary: `Peirce`
- Implementation support: `Ampere`

## Goal

Use UI UX Pro Max as design intelligence for both the public Scanner and the Cerebro HUD, while keeping each surface true to its job.

## Shared Design Language

### Visual direction

- Tactical, premium, market-intelligence interface
- Dark atmospheric base with selective luminous accents
- Strong information hierarchy over decorative density

### Design primitives

- Glassmorphism only where it improves layering and focus
- Bento layouts for grouped intelligence blocks
- HUD motifs for controls, telemetry, and tactical emphasis

### Shared tokens to define

- typography scale
- color roles
- panel surfaces
- border/glow rules
- spacing system
- status/alert colors

## Surface Split

### Public Scanner

- Job: rank, explain, and publish actionable daily intelligence
- Tone: premium and legible first
- Needs:
  - fast scanability
  - freshness visibility
  - strong list/table/card hierarchy
  - confidence without overwhelming the visitor

### Cerebro HUD

- Job: operator console for exploration and synthesis
- Tone: more immersive and dense than the public Scanner
- Needs:
  - graph and telemetry support
  - active spark/event focus
  - higher information density
  - stronger tactical/HUD personality

## Rules

- Do not let the public Scanner become as visually dense as the HUD
- Do not use effects that reduce legibility of price, score, or timing data
- Shared design system, different intensity per surface

## First Deliverables

1. token draft for shared surfaces
2. Scanner homepage layout concept
3. Cerebro HUD panel hierarchy refresh
4. list of what UI UX Pro Max should influence directly in code

## Success Test

Someone looking at both products should immediately feel they come from the same system, but also instantly understand that:

- the Scanner is the public intelligence surface
- Cerebro is the operator console
