How we exit

Every position is governed by:
  • A profit target (partial close on hit, trail rest)
  • A hard stop (capital preservation)
  • A trailing stop (only arms after the position is held overnight — this keeps us PDT-rule-safe)
  • A time stop (close after a fixed number of trading days regardless of P/L)

The exact thresholds are internal — they tune via the closed-loop validator. What matters is that every position has 4 exit triggers, every one of them automatic.

How we size

Position size scales with realized historical edge on the published score≥15 cohort. When edge is positive and stable, sizing scales up. When edge is negative or noisy, sizing falls back to a fixed floor. This is governed by walk-forward (out-of-sample) performance, not the in-sample tuning window.

Halt conditions (hard-coded)

  • Daily loss exceeds threshold → new entries auto-halt; exits continue
  • Pattern-day-trade count nears the regulatory ceiling → entries halt
  • Manual halt file → entries halt; exits ALWAYS continue regardless
  • Tradier API error → cycle skipped, retried next cron tick
