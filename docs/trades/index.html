How this works

Three steps. Open source where it matters.

01 · Signals

Scanner + cross-border feed

The agent reads combined_priority.csv and the cross-border STRONG/TRADE setups. Top 5 candidates by composite score, US-listed only on the first cut.

/scanner →
02 · Sizing

$50 cap, 5 max open

Each signal gets at most $50 of position. Daily-loss kill switch at −$200 — agent halts itself rather than chase. Idempotent client-order-IDs prevent re-fires.

/cross-border →
03 · Execution

Alpaca paper bracket

Market order with attached take-profit (+5%) and stop-loss (−3%). Fills, fills, and misses all logged to /data/alpaca_orders.json for public audit.

/status →
